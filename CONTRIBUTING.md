# Contributing to dft2dxf

Thank you for contributing.

## Development setup

1. Install Rust 1.88+ and platform linker tooling.
2. Clone the repository.
3. Run `cargo build`, `cargo test`, `cargo clippy --all-targets`, and `cargo fmt --all`.

## Commit style

Use semantic prefixes:

- `feat:` new functionality
- `fix:` bug fixes
- `refactor:` internal restructuring
- `test:` test-only changes
- `docs:` documentation
- `chore:` tooling and maintenance

## Fixture policy

- Never commit proprietary customer `.dft` files without permission.
- Document provenance for every committed fixture.
- Prefer synthetic fixtures generated by `dft2dxf-testkit`.

## Pull request checklist

- [ ] Tests added or updated
- [ ] Public APIs documented
- [ ] Limits and error handling considered for hostile input
- [ ] README or docs updated for user-visible changes
